# 撮影用QR決済モックアプリ（PWA）要件定義 ＆ 実装指示書

## 1. プロジェクト概要
映像作品の撮影現場で「小道具（プロップ）」として使用する、スマートフォン向けのQR決済モックアプリ（Webアプリ）。
実際の決済機能やQR解析処理は一切持たず、「カメラ映えするUI」と「現場でのNGテイク削減（確実な画面遷移）」に特化した専用設計とする。

## 2. 技術スタック
- **フレームワーク**: Next.js (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **アニメーション**: Framer Motion
- **状態管理**: Zustand
- **アイコン**: Lucide React
- **カメラ制御**: Web API (`navigator.mediaDevices.getUserMedia`)

## 3. コア要件・制約事項（※重要）
撮影現場特有の要件を満たすため、以下の制約を厳守して実装すること。

- **スマホ特化**: レスポンシブ（PC/タブレット対応）は不要。モバイルサイズでの表示を大前提とする。
- **ブラウザ挙動の完全排除**: ネイティブアプリ感を損なわないよう、以下の設定を必ず含めること。
  - ピンチズーム禁止: `<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=0" />`
  - スクロールバウンス禁止: `body { overscroll-behavior: none; touch-action: pan-y; }`
- **画面高さの最適化**: アドレスバーによるレイアウト崩れを防ぐため、高さは `100vh` ではなく `100dvh` を使用する。
- **PWA対応**: ホーム画面に追加してフルスクリーンで起動できるよう、最低限の `manifest.json` を用意する。

## 4. 状態管理 (Zustand Store)
以下の状態をグローバルに管理し、全画面で共有・同期する。
- `themeColor` (string): アプリのテーマカラー（HEX値。デフォルト: `#ff0033`）
- `amount` (number): 支払い金額（デフォルト: 1500）
- `shopName` (string): 店舗名（デフォルト: 'NonTurn Cafe'）

## 5. 画面構成と機能要件

### A. スキャン画面 (`app/page.tsx`)
- **背面カメラの常時起動**: 最背面に `video` タグを絶対配置し、デバイスの背面カメラ映像を全画面 (`object-cover`) でストリーミング表示する。
- **UIオーバーレイ (前面レイヤー)**:
  - ヘッダー帯（`themeColor`適用、「コード支払い」等のダミータイトル）
  - 画面全体に半透明の黒マスク（中央の正方形部分のみくり抜いて透明にする）
  - 中央の読み取り枠（四隅に `themeColor` のL字太字ボーダーを配置）
  - 枠内を上下に往復する「スキャンライン（レーザー光）」アニメーション（Framer Motionを使用）
  - ダミーのボトムナビゲーションバー（ホーム、スキャン、履歴等のアイコン。タップしても遷移しない）
- **【現場用チート機能】**: 画面内の「どこをタップしても（設定ドロワー領域を除く）」即座に B. 決済完了画面 (`/success`) へルーター遷移 (`router.push`) する仕様とする。

### B. 隠し設定ドロワー (Bottom Sheetコンポーネント)
- **トリガー**: スキャン画面の「右下の特定の余白（透明な領域）」を **1秒以内に3回連続タップ** した場合にのみ出現する。
- **UI**: 画面下部からスライドイン（Framer Motion）するハーフモーダルUI。ドロワーの背景クリックで閉じる機能も実装する。
- **機能**:
   - `themeColor` の変更（カラーピッカー `type="color"` と HEXテキスト入力の併用）
  - `amount`（決済金額）の変更（数値入力）
  - `shopName`（店舗名）の変更（テキスト入力）
  - これらの変更はZustandのStoreに即時反映させ、スキャン画面のUI（レーザーやヘッダー）にもリアルタイムで適用されること。

### C. 決済完了画面 (`app/success/page.tsx`)
- **UI**:
  - 「ピロン♪」と鳴ったかのような爽快なアニメーション。
  - 画面中央に大きく「チェックマーク」のアニメーション（Framer Motionでスケールイン＆パス描画）。チェックマークの背景円などに `themeColor` を適用。
  - Zustandから取得した `amount` を大きく表示（例: `¥1,500`）。
  - Zustandから取得した `shopName` を表示（例: `支払い先: NonTurn Cafe`）。
- **機能**:
  - 画面下部に「スキャンを続ける（戻る）」ボタンを配置し、スキャン画面 (`/`) へ戻れるようにする。

## 6. Claude Code への実装ステップ指示

1. Next.js (App Router, TypeScript, Tailwind) 環境を構築する。
2. 必要なパッケージ (`framer-motion`, `zustand`, `lucide-react`) をインストールする。
3. `src/store/useMockStore.ts` を作成し、状態管理を実装する。
4. `app/globals.css` と `app/layout.tsx` に、スマホネイティブ化のためのviewport設定とCSSハックを記述する。
5. `src/components/SettingsDrawer.tsx` を作成し、3回タップの隠しコマンドとUIを実装する。
6. `app/page.tsx` にカメラストリーミング、UIオーバーレイ、レーザーアニメーションを実装し、画面タップでの遷移ロジックを組み込む。
7. `app/success/page.tsx` を作成し、決済完了のアニメーションとデータ表示を実装する。
8. 動作確認後、不要なテストファイルは生成しないこと。コードはシンプルかつメンテナンスしやすく保つこと。

## 7. 撮影現場特有のエッジケース対策（必須要件）

AIエージェントは以下のエッジケースを考慮し、必ず対策コードを含めること。

### 7.1 状態の永続化 (Zustand Persist)
現場で設定した値がブラウザの再起動やリロードで消失しないよう、ZustandのStoreには必ず `persist` ミドルウェアを適用し、`localStorage` に状態（`themeColor`, `amount`, `shopName`）を保存すること。

### 7.2 効果音の再生とAutoplay Policy回避
決済完了画面での「ピロン♪」という効果音（`/public/success.mp3` を想定）を確実に鳴らすため、遷移前の「スキャン画面のチートタップ処理」のイベント内でAudioインスタンスを生成/再生開始するか、ユーザーインタラクションを起点とした確実な再生ロジックを実装すること。

### 7.3 iOS向けPWAの完全擬態（Apple Touch Icon）
iPhoneでホーム画面に追加した際、ネイティブアプリとして完璧に擬態できるよう、`app/layout.tsx` のメタデータ、または `app/manifest.ts` 等において、`apple-touch-icon` を適切に設定する記述を含めること（アイコン画像自体はダミーのプレースホルダーで構わない）。

### 7.4 カメラエラー時のフォールバックUI
デバイスのカメラ権限が拒否された場合や、`getUserMedia` がエラーを吐いた場合にアプリがクラッシュ・フリーズしないこと。エラーを捕捉（catch）し、カメラ映像の代わりに「ダークグレーの背景（#333など）」を表示させ、その上のUIレイヤーやタップ遷移機能はそのまま正常に動作させ続けること。
